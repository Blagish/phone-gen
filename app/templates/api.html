API сайта

1. Синтаксис запроса

Чтобы обратиться к методу API, Вам необходимо выполнить GET запрос такого вида:
https://site.name/METHOD_NAME?PARAMETERS

METHOD_NAME (Обязательно) - Имя метода, к которому производится обращение.
PARAMETERS (Опционально) - Входные параметры соответствующего метода, пары name=value, разделенные амперсандом (&). Список параметров указан в соответствии с каждым методом.

2. Имеющиеся методы

Метод get_info
Возвращает информацию о номере телефона в формате JSON.
Параметры:
  • phone - номер телефона. Строка из 10 или 11 чисел без лишних символов. Если в строке 11 символов, предполагается, что первый символ - это международный код, 7 или 8. Он автоматически отбрасывается.
Пример запроса:
https://site.name/get_info?phone=3012115678
Пример ответа:
{"code": "ok","provider": "ПАО "Ростелеком"","region": "Республика Бурятия","location": "г. Улан-Удэ"}
Параметры ответа:
  • code - код результата. Успешно - "ok", запрос не является номером - "nan" (Not A Number), номер отсутствует в базе данны - "ne" (Not Exist).
  • provider - Наименование провайдера.
  • region - Наименование региона.
  • location - Более точное место, к которому привязан номер. В базе данных полная локация записывается как %location%|%region%

Метод get_phones
Возвращает список номеров, соответствующий критериям.
Параметры:
  • region_id - идентификационный номер региона в соответствии со списком, представленным на сайте.
  • provider_id - идентификационный номер провайдера в соответствии со списком, представленным на сайте.
  • count - число запрашиваемых телефонных номеров для генерации.
Пример запроса:
https://site.name/get_phones?region_id=59&provider_id=58&count=10
Пример ответа:
9580498172
9580497945
3012201432
3012200995
9580498789
9580495634
3012201643
9580492048
3012201637
3012200832
Если с заданными параметрами невозможно сгенерировать номера телефонов, выдаст ошибку.
